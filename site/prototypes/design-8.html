¶ The Typographic Analogy

The Invisible Foundation

In typography, the baseline is the invisible line upon which most letters sit and below which descenders extend. It is the fundamental constraint that provides rhythm, order, and legibility to text. Without it, language dissolves into chaos.

In the era of AI-generated code, the Baseline programming language serves the same function. It provides the rigid, mathematical structure necessary to constrain the probabilistic output of Large Language Models, turning unpredictable generation into verifiable software.

I. The Type is the Spec

Types encode enough information that correctness is verified, tests are generated, and documentation is accurate. We parse; we do not validate.

II. Effects are Data

Side effects are explicit capabilities, tracked and mockable. Pure functions are the default; effects are opt-in permissions.

III. LLM-Native

Designed for machine generation. Syntax is unambiguous, errors are structured JSON, and the verification loop is automated.

Principles in Practice

Constraining the Solution Space

Traditional languages offer unlimited ways to err. Baseline uses refinement types to mathematically constrain boundaries, eliminating entire classes of hallucinated bugs.

// The type IS the specification.
type Port = Int where 1 <= self <= 65535

listen : Port -> {Net} Server
listen = |port|
  // port is GUARANTEED to be 1-65535 here.
  // No defensive coding needed.
  ...
Fig 1. Refinement types as boundary constraints.

Capability-Based Security

Agents cannot hallucinate I/O. Side effects are treated as data capabilities. An agent without the Fs effect capability physically cannot access the filesystem.

agent_task! : Input -> {Log, Db.read} Output
agent_task! = |input|
  Log.info!("Processing ${input}")
  let data = Db.query!("SELECT * FROM items")

  // COMPILE ERROR: Fs not in effect set
