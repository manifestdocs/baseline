What Other Languages Got Wrong

01

The Billion-Dollar Mistake, Repeated

Languages keep making the same mistakes: null references, unchecked exceptions, implicit side effects. JavaScript throws at runtime. Python hopes you wrote tests. Go returns (value, error) and prays you checked both. Java's checked exceptions were the right idea with the wrong execution.

Baseline's answer: Result<T, E> and Option<T> are the only error paths. Pattern matching is exhaustive. No null. No exceptions. No silent failures.
02

The Colored Function Problem

async/await splits every codebase in two. Node.js callback hell evolved into promise hell evolved into async/await hell -- the colors just changed. Rust's async is powerful but infectious. Go solved it with goroutines but lost type safety on channels.

Baseline's answer: Async is an effect, not function coloring. {Async} in the type signature. The same function works sync or async depending on the handler. No .await, no Promise<T>, no two worlds. Direct style, powered by algebraic effects.
03

Side Effects Are Security Holes

Any function can do anything in most languages. A dependency can read your filesystem, phone home, or delete files. npm packages have proven this isn't theoretical. Python's import os is available everywhere. Even Rust's safety guarantees don't cover side effects.

Baseline's answer: Effects are capabilities declared in type signatures. {Console, Http} means a function can print and make HTTP calls -- nothing else. The effect system is a compiler-enforced security sandbox, especially critical when LLM agents are generating and executing code autonomously.

Concurrency Without Complexity

Structured Concurrency

Fibers are lightweight (2KB initial stack), scoped, and automatically cleaned up. Parent tasks wait for children. If any fiber panics, siblings are cancelled. No fire-and-forget. No orphaned tasks. No resource cleanup nightmares. 

process! : List<Item> -> {Async} List<Result>
process! = |items|
  scope! |s|
    for item in items do
      s.spawn!(|| process_item!(item))
    // All fibers complete or cancel together

Parallel by Default

Fan out, collect, race. parallel! runs tasks concurrently and collects all results. race! returns the first to complete and cancels the rest. No thread pool configuration. No executor setup. Just declare the Async effect. 

let results = parallel!([
  || fetch_users!(),
  || fetch_posts!(),
  || fetch_comments!(),
])

let fastest = race!([
  || fetch_from_primary!(),
  || fetch_from_replica!(),
])

Channels with Backpressure

Bounded channels prevent producer-consumer imbalance. Senders block when the buffer is full. Select across multiple channels with timeouts. The type system tracks it all. 

let (tx, rx) = Channel.bounded<Int>(100)

spawn! ||
  for i in 1..1000 do
    tx.send!(i)
  tx.close!()

select!
  recv!(rx1) as msg -> handle_a!(msg)
  recv!(rx2) as msg -> handle_b!(msg)
  after 5.seconds   -> timeout!()

Purpose-Built for the Agentic Age

When Machines Write the Code, the Language Matters More

LLMs are writing more code every day. Most languages were designed for humans writing code by hand. When an LLM generates JavaScript, Python, or even Rust, it navigates ambiguity, implicit behavior, and multiple valid ways to express the same thing. Baseline constrains the solution space. One way to handle errors (Result). One way to express side effects (the effect set). One way to define APIs (type-checked specifications). The type signature IS the spec -- if the LLM gets the type right, the implementation is constrained to be correct.

The effect system is a sandbox. When an AI agent generates Baseline code, you know exactly what it can do by reading the type signature. {Log, Db.read} means it can log and query -- not delete files, not make network calls, not access the filesystem. This is not a convention. It is compiler-enforced.

Structured JSON output from the compiler enables automated self-correction loops: LLM generates code, the compiler checks it, structured errors flow back, the LLM fixes the issues, and the cycle repeats until the types check. No test suite required for basic correctness. The type system does the work.

Everything You Need. Nothing You Don't.

Baseline ships the features that matter for building reliable connected systems.

Refinement Types

Encode value constraints in types. Port = Int where 1 <= self <= 65535. Proven at compile time.

Algebraic Effects

Track every side effect in type signatures. The compiler enforces what is permitted.

Pattern Matching

Exhaustive by default. Every branch handled. No fall-through, no missed cases.

Pipe Operator

|> chains transformations left to right. Clear data flow, no nesting.

Sum Types

Model domain variants precisely. Result, Option, and your own discriminated unions.

Records

Structural typing with named fields. Dot access. Type-safe construction.

First-Class HTTP

Routes, parameters, status codes -- all type-checked. No YAML sidecars.

Structured Concurrency

Scoped fibers, parallel combinators, bounded channels. No leaked tasks.

Region Memory

Arena allocation for request-scoped data. O(1) deallocation. No GC pauses.

WebAssembly

Primary compilation target. Deploy to edge, browser, or server.

Security Sandbox

Effect system as capability boundary. Safe for AI-generated code execution.

JSON Diagnostics

Structured compiler output. Enables automated LLM self-correction loops.
