Baseline

A strongly typed, effect-based programming language. Code, like memory, builds meaning layer by layer. Each type a truth. Each effect a boundary. Each line a foundation.

Layer One

The Type Is the Spec

"The first memory. The foundation on which everything is built."

In Baseline, types are not annotations -- they are proofs. A Port is not an Int. It is an Int proven to be within bounds. The compiler does not trust. It verifies.

Refinement Types types.bl 
type Port = Int where 1 <= self <= 65535

listen : Port -> {Net} Server
listen = |port|
  // port is GUARANTEED to be 1-65535 here
  // No defensive coding needed
  ...
Layer Two

Effects Are Data

"Deeper now. What you are permitted to touch."

Every side effect is an explicit capability. A function that logs declares it. A function that reads a database declares it. Attempt to access something undeclared? The compiler stops you. The effect system is a security boundary.

Effect-Based Security sandbox.bl 
agent_task! : Input -> {Log, Db.read} Output
agent_task! = |input|
  Log.info!("Processing ${input}")
  let data = Db.query!("SELECT * FROM items")
  // Fs.delete!("important.txt")
  // COMPILE ERROR: Fs not in effect set
  transform(data)
Layer Three

LLM-Native

"The deepest layer. The one that explains why."

Baseline is designed for an era where machines write code and humans verify it. Unambiguous syntax means LLMs generate correct code more often. The effect system means AI agents can only access what you explicitly allow. Trust, but verify -- at compile time.

Hello World hello.bl 
@prelude(script)

greet : String -> String
greet = |name| "Hello, ${name}!"

main! : () -> {Console} ()
main! = {
  Console.print!(greet("World"))
}
Composition

Data Flows

Pipe operators compose transformations naturally. Each step is a function. The data flows left to right, building meaning with each transformation.

Pipe Composition pipes.bl 
let chained = nums
  |> |xs| List.map(xs, |x| x * 2)
  |> |xs| List.filter(xs, is_even)
  |> |xs| List.fold(xs, 0, |acc, x| acc + x)
What Has Been Built

The Full Excavation

  • Refinement Types

    Value constraints encoded in the type system and proven at compile time. No runtime checks needed.

  • Algebraic Effects

    Side effects tracked as explicit capabilities in type signatures. Console, Http, Fs -- declared and enforced.

  • Pattern Matching

    Exhaustive match expressions for sum types. Every case handled. No surprises at runtime.

  • Pipe Operator

    Left-to-right data transformation with |>. Composable, readable, predictable.

  • ML-Style Syntax

    name : Sig = body. Every function declares its contract. Types and effects are front and center.

  • Sum Types

    Discriminated unions for domain modeling. Result, Option, and custom variants for precise data shapes.

  • Records

    Structural data types with named fields, dot access, and type-safe construction.

  • Security Sandbox

    The effect system doubles as a capability sandbox. AI agents and plugins access only what you grant.
